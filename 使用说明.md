## 一、部署准备（教师操作）

### 1. 环境要求

- 教师机操作系统：Windows Server 2016 或 Windows 10/11
- 已安装 Python 3.9 或以上版本（[下载地址](https://www.python.org/downloads/)）
- 教师机与学生机处于同一局域网

### 2. 启动服务器

双击项目文件夹中的 `start.bat`，脚本会自动完成以下操作：

1. 安装所需 Python 依赖包
2. 检查是否存在管理员账号，若不存在则引导创建
3. 启动 Web 服务器（默认端口 8080）

启动成功后，命令行窗口显示：

```
服务器启动中...
学生访问地址: http://教师机IP:8080
教师后台地址: http://教师机IP:8080/admin/login
按 Ctrl+C 停止服务器
```

### 3. 关闭服务器

在服务器命令行窗口中按 **Ctrl + C** 即可停止。

### 4. 获取教师机 IP 地址

在教师机上打开命令提示符，输入 `ipconfig`，找到"IPv4 地址"，例如 `192.168.1.100`。学生访问的完整地址为：

```
http://192.168.1.100:8080
```

> **建议**：将上述网址创建为浏览器书签或桌面快捷方式，通过极域电子教室等软件下发到所有学生机桌面，方便学生一键访问。

### 5. 开机自启（可选）

如需每次开机自动启动服务器，可将 `start.bat` 的快捷方式放入 Windows 启动文件夹（`shell:startup`）。

### 6. 文件存储说明

系统会在教师机上自动创建两个存储目录：

| 目录 | 用途 |
|------|------|
| `D:\ClassUploads\` | 图片和视频文件的存储位置，按学生姓名分文件夹 |
| `D:\ClassWebRoot\` | 网页项目的存储位置，按学生姓名分文件夹 |

## 二、教师端使用说明

教师端拥有独立的管理后台，用于审核学生账号、管理上传文件、审核网页项目。

### 2.1 登录管理后台

1. 浏览器访问 `http://教师机IP:8080/admin/login`
2. 输入管理员账号和密码（首次启动时通过 `create_admin.py` 创建）
3. 点击"登录"

### 2.2 管理后台首页（仪表盘）

登录后进入仪表盘，顶部显示统计卡片：

| 统计项 | 含义 |
|--------|------|
| 待审核账号 | 新注册等待审批的学生数量 |
| 已通过学生 | 已批准可正常使用的学生数量 |
| 待审核作品 | 学生上传但尚未审核的图片/视频数量 |
| 已发布作品 | 已审核通过的图片/视频数量 |
| 已发布网页 | 已审核通过的网页项目数量 |
| 待审核网页 | 学生上传但尚未审核的网页项目数量 |
| 已发布占用空间 | 已通过文件占用的磁盘空间 |

功能入口卡片：

- **账号审核** — 审批学生注册申请
- **文件管理** — 查看和审核所有上传的图片/视频
- **网页项目管理** — 查看和审核所有网页项目

### 2.3 账号审核

1. 点击"账号审核"进入待审核列表
2. 列表显示每个待审核学生的姓名和注册时间
3. 对每个学生，选择操作：
   - **通过**：批准注册，系统自动在 `D:\ClassUploads\` 和 `D:\ClassWebRoot\` 下创建以该学生姓名命名的文件夹，学生即刻可以登录和使用所有功能
   - **拒绝**：拒绝注册，学生将无法登录

> 审核通过后，学生立即获得上传权限，无需额外配置。

### 2.4 文件管理（图片/视频看板）

1. 点击"文件管理"进入文件看板
2. 可通过顶部下拉菜单按**学生姓名**或**审核状态**（待审核/已通过/已拒绝）筛选

表格字段说明：

| 列 | 说明 |
|----|------|
| 预览 | 图片显示缩略图，视频显示"视频"标签。点击缩略图可全屏预览 |
| 文件名 | 学生上传的原始文件名 |
| 上传者 | 上传该文件的学生姓名 |
| 上传时间 | 精确到分钟的上传时间 |
| 类型 | "图片"或"视频" |
| 大小 | 文件大小（MB） |
| 审核状态 | **待审核**（黄色）/ **已通过**（绿色）/ **已拒绝**（红色） |
| 操作 | 见下方说明 |

**操作按钮随审核状态变化**：

| 文件状态 | 可用操作 |
|----------|----------|
| 待审核 | **通过** — 文件对全班可见；**拒绝** — 文件被打回，学生端显示"作品被打回" |
| 已通过 | **预览** — 弹窗查看/播放；**删除** — 永久删除文件 |
| 已拒绝 | **改为通过** — 改变决定，文件变更为已通过；**删除** — 永久删除 |

> 教师删除文件为永久删除，不可恢复。教师也可直接在 Windows 资源管理器中打开 `D:\ClassUploads\` 文件夹查看所有作品。

### 2.5 网页项目管理

1. 在仪表盘点击"网页项目管理"进入网页看板
2. 可通过顶部下拉菜单按**审核状态**筛选

表格字段说明：

| 列 | 说明 |
|----|------|
| 项目名称 | 学生上传 ZIP 时的项目名称（取文件名，不含 .zip 后缀） |
| 部署学生 | 上传该网页项目的学生姓名 |
| 部署时间 | 上传并解压完成的时间 |
| 解压大小 | ZIP 解压后的总文件大小 |
| 访问地址 | 网页的公开访问 URL（仅已通过项目显示） |
| 审核状态 | 待审核 / 已通过 / 已拒绝 |
| 操作 | 见下方说明 |

**操作按钮随审核状态变化**：

| 项目状态 | 可用操作 |
|----------|----------|
| 待审核 | **通过** — 网页对全班开放；**拒绝** — 学生端显示"作品被打回" |
| 已通过 | **预览** — 新标签页打开网页；**下线删除** — 永久删除 |
| 已拒绝 | **改为通过** — 改变决定；**删除** — 永久删除 |

> 点击"预览"会在新标签页中打开学生制作的网页，方便教师检查内容是否合规。

### 2.6 退出管理后台

点击顶部导航栏右侧的"退出"按钮即可退出。

## 三、学生端使用说明

### 3.1 打开网站

在浏览器地址栏输入教师公布的网址（如 `http://192.168.1.100:8080`），进入登录页面。

### 3.2 注册账号

1. 在登录页点击"立即注册"
2. 输入自己的**真实姓名**（作为账号唯一标识）
3. 输入密码（至少 4 位）并确认密码
4. 点击"注册"
5. 看到提示"注册成功！请等待教师审核通过后登录"

> **注意**：注册后不能立即登录，需等待教师审核通过。

### 3.3 登录

1. 在登录页输入姓名和密码
2. 点击"登录"
3. 可能遇到的情况：

| 情况 | 提示信息 | 处理方式 |
|------|----------|----------|
| 账号/密码错误 | "姓名或密码错误" | 检查输入是否正确 |
| 还未审核 | "账号正在等待教师审核" | 举手告知老师审核 |
| 审核被拒 | "账号审核未通过" | 联系老师了解原因 |
| 审核通过 | "欢迎回来，XXX！" | 自动进入主页 |

### 3.4 主页布局

登录成功后进入主页，顶部导航栏显示"课堂作品平台"、你的姓名和退出按钮。

页面包含三个 Tab 页签：

---

#### Tab 1：「我的作品」— 图片/视频上传和管理

**上传文件**：
- 点击虚线框内的云朵区域，或者直接将文件拖拽到虚线框内
- 支持的文件类型：图片（JPG、PNG、GIF 等）、视频（MP4、MOV 等）
- 单文件最大 100MB
- 上传过程中显示进度条百分比
- 上传成功后提示"作品已提交，请等待教师审核通过后展示给同学"

**文件审核状态**：

| 标签 | 含义 |
|------|------|
| ⏳ 待审核 | 文件已上传成功，等待教师审核 |
| ✅ 已通过 | 教师已审核通过，全班同学可见 |
| ❌ 作品被打回 | 教师拒绝了该文件，仅自己可见 |

**管理文件**：
- 文件以网格卡片形式展示，图片直接显示缩略图，视频显示播放图标
- 点击图片或视频卡片可弹窗全屏预览/播放
- 鼠标悬停在文件卡片上，右上角出现删除按钮（🗑），点击可删除自己的文件
- 任何审核状态的文件都可以删除

---

#### Tab 2：「我的网页」— 网页项目上传和管理

详见 [四、网页托管功能](#四网页托管功能)。

---

#### Tab 3：「同学作品」— 浏览全班作品

**同学文件夹**（图片/视频）：
- 以文件夹卡片形式展示每位有已通过作品的同学
- 每个卡片显示：同学姓名、最新图片作为封面（若无则显示 📁）、已通过文件总数
- 点击卡片进入该同学的作品列表，点击图片/视频可弹窗预览

**网页作品画廊**（网页项目）：
- 在文件夹列表下方以独立区域展示
- 每张卡片显示橙色 `</>` 代码图标、项目名称、作者姓名、部署日期
- 点击卡片会在**新标签页**中打开该同学制作的网页
- 其他同学可在此浏览、体验彼此的网页交互作品

> **安全提示**：所有作品都需要教师审核通过后才会出现在"同学作品"页。视频播放使用 HTML5 原生播放器，点击 ▶ 按钮即可播放，不会一次性加载完整视频，节省网络带宽。

### 3.5 退出

点击顶部导航栏的"退出"按钮退出登录。

## 四、网页托管功能

此功能允许学生将自制的网页作品（HTML/CSS/JS）打包上传到教师机，部署后可互相访问。

### 4.1 学生端：上传网页项目

**准备网页**：
1. 在自己的电脑上创建一个文件夹，放入所有网页文件
2. 文件夹内**必须包含一个 `index.html` 文件**作为首页入口
3. 将该文件夹压缩为 `.zip` 格式

> 例如：文件夹名为 `myweb`，里面有 `index.html`、`style.css`、图片等。将整个 `myweb` 文件夹压缩为 `myweb.zip`。

**上传部署**：
1. 在主页切换到「我的网页」Tab
2. 点击橙色虚线框区域（或拖拽 ZIP 文件到框内）
3. 选择你的 `.zip` 文件，系统会自动上传、解压、部署
4. 等待进度条完成，提示"网页部署成功！请等待教师审核通过后对全班开放"

**限制说明**：

| 限制项 | 值 |
|--------|-----|
| 仅支持格式 | `.zip` |
| 单 ZIP 大小 | 最大 50MB |
| 个人总大小 | 所有项目解压后合计不超过 50MB |
| 必含文件 | `index.html`（或 `index.htm`） |

**管理已部署的网页**：

每个网页项目显示为一个列表卡片，包含项目名称、解压大小和审核状态。三个操作按钮：

| 按钮 | 功能 |
|------|------|
| **访问预览** | 仅已通过项目显示，在新标签页打开你的网页 |
| **更新** | 重新上传同名 ZIP 包，覆盖旧版本（需重新审核） |
| **删除** | 卸载该网页项目，服务器上的文件被永久删除 |

### 4.2 学生端：访问同学的网页

1. 切换到「同学作品」Tab
2. 向下滚动到"网页作品画廊"区域
3. 点击任意橙色 `</>` 卡片，在新标签页中打开该同学的网页
4. 可在网页中进行交互、浏览，就像访问真正的网站一样

### 4.3 教师端：管理网页项目

详见 [2.5 网页项目管理](#25-网页项目管理)。

### 4.4 访问地址格式

每个已审核通过的网页项目拥有唯一的 URL：

```
http://教师机IP:8080/web/学生姓名/项目标识/
```

例如：`http://192.168.1.100:8080/web/张三/a1b2c3d4/`

> 网页项目之间完全隔离，无法访问其他学生项目中的文件或服务器其他目录。

## 五、业务处理流程

### 流程一：学生首次使用全流程

```
学生注册 → 教师审核账号 → 学生登录 → 上传作品/网页 → 教师审核作品 → 全班浏览
```

1. **注册**：学生在浏览器输入教师机 IP，进入注册页，填写姓名和密码
2. **等待**：系统提示"请等待教师审核"
3. **账号审核**：教师在管理后台→"账号审核"，找到该学生，点击"通过"
4. **登录**：学生用新账号登录，进入主页
5. **上传**：学生在「我的作品」上传图片/视频，或在「我的网页」上传网页项目
6. **作品审核**：教师在"文件管理"或"网页项目管理"中审核学生作品
7. **展示**：审核通过后，全班同学在「同学作品」页可看到

### 流程二：作品审核流程（图片/视频/网页通用）

```
学生上传 → 文件状态="待审核"
              ↓
      教师后台查看 → 预览内容
              ↓
    ┌─────────┴─────────┐
    ↓                   ↓
 点击"通过"         点击"拒绝"
    ↓                   ↓
 全班可见          仅学生本人可见
                   红色标签"作品被打回"
```

> 学生可在「我的作品」或「我的网页」中随时查看自己每个文件/项目的审核状态。被拒绝的作品只有上传者本人和教师可见。

### 流程三：新上课时的快速启动

```
打开教师机 → 双击 start.bat → 确认服务器启动 → 学生访问
```

### 流程四：数据备份

教师可定期备份以下数据：

| 备份内容 | 路径 | 说明 |
|----------|------|------|
| 数据库文件 | `项目目录\data.db` | 所有用户、文件和网页项目记录 |
| 图片/视频文件 | `D:\ClassUploads\` | 按学生姓名分文件夹 |
| 网页项目文件 | `D:\ClassWebRoot\` | 按学生姓名分文件夹 |

建议将上述内容定期复制到 U 盘或其他备份位置。

## 六、常见问题

### Q1：学生忘记密码怎么办？

在教师机上运行以下命令重置学生密码（将 `学生姓名` 和 `新密码` 替换为实际值）：

```bash
python -c "from app import create_app; from models import db, User; import bcrypt; app=create_app(); ctx=app.app_context(); ctx.__enter__(); u=User.query.filter_by(username='学生姓名',role='student').first(); u.password_hash=bcrypt.hashpw('新密码'.encode(),bcrypt.gensalt()).decode(); db.session.commit(); print('密码已重置')"
```

### Q2：学生注册时提示"该姓名已被注册"？

说明已有同名学生注册。请确认是否重复注册，或联系教师确认情况。

### Q3：学生上传后为什么同学看不到？

所有作品上传后默认状态为"待审核"，需要教师在后台审核通过后才会在「同学作品」页展示。请提醒教师去后台处理。

### Q4：网页上传失败怎么办？

常见原因：
- ZIP 包内没有 `index.html` 文件
- ZIP 文件超过 50MB
- 解压后总大小超过个人配额（50MB）
- 项目名称包含特殊字符（`\ / : * ? " < > |`）
- ZIP 包内包含非法路径（如 `../` 开头）

### Q5：服务器启动报错"端口被占用"？

修改 `config.py` 中的 `PORT` 为其他端口号（如 8081），重新启动。同时告知学生新的端口号。

### Q6：学生机无法访问教师机？

1. 检查教师机和学生机是否连接同一局域网
2. 检查教师机防火墙是否允许 8080 端口入站
3. 在教师机上打开 `http://localhost:8080` 确认服务正常运行

### Q7：可以更换文件存储位置吗？

修改 `config.py` 中对应的路径变量：

| 变量 | 用途 | 默认值 |
|------|------|--------|
| `UPLOAD_ROOT` | 图片/视频存储 | `D:\ClassUploads` |
| `WEB_ROOT` | 网页项目存储 | `D:\ClassWebRoot` |

修改后重启服务器即可，已上传的文件需手动迁移到新路径。

### Q8：文件上传或网页项目大小限制可以调整吗？

修改 `config.py` 中对应的变量：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MAX_CONTENT_LENGTH` | 100MB | 图片/视频单文件最大 |
| `MAX_ZIP_SIZE` | 50MB | 网页 ZIP 压缩包最大 |
| `MAX_TOTAL_WEB_SIZE` | 50MB | 单学生网页项目总大小上限 |

> 所有大小单位均为字节，修改后重启服务器生效。

### Q9：教师审核学生注册时，系统自动做了哪些事？

点击"通过"后，系统自动：
1. 将学生账号状态从"待审核"改为"已通过"
2. 在 `D:\ClassUploads\<学生姓名>\` 下创建个人文件夹
3. 在 `D:\ClassWebRoot\<学生姓名>\` 下创建网页托管文件夹

学生即刻可登录、上传图片视频、部署网页项目。

### Q10：学生可以同时上传多个文件或网页吗？

可以。学生可以多次上传图片/视频和网页项目，每次上传都会进入"待审核"状态，等待教师逐一审核。

# 需求变更文档：作品上传审核与学生删除

## 变更一：作品上传审核机制

### 变更背景

原有流程中学生上传作品后立即公开发布，教师无法在上传环节进行内容把关。为满足课堂教学管理需求，增加教师审核环节。

### 业务流程

```
学生上传作品 → 文件保存，状态=「待审核」
                  ↓
         学生"我的作品"页：文件卡片显示「待审核」标签，文件暂不对其他同学可见
                  ↓
         教师进入后台"文件管理" → 查看待审核文件 → 预览内容
                  ↓
       ┌──────────┴──────────┐
       ↓                     ↓
   点击「通过」           点击「拒绝」
       ↓                     ↓
 文件状态=「已通过」     文件状态=「已拒绝」
 对全班同学可见          学生对文件卡片显示
                         「作品被打回」红色标签
                         仅学生本人和教师可见
```

### 数据变更

`FileRecord` 表新增字段：

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| status | String(10) | 'pending' | 'pending'=待审核 / 'approved'=已通过 / 'rejected'=已拒绝 |

### 各角色可见性规则

| 文件状态 | 上传者本人（学生） | 其他学生 | 教师 |
|----------|-------------------|----------|------|
| pending（待审核） | 可见，带"待审核"标签 | 不可见 | 可见，可操作通过/拒绝 |
| approved（已通过） | 可见，带"已通过"标签 | 可见 | 可见，可删除 |
| rejected（已拒绝） | 可见，带红色"作品被打回"标签 | 不可见 | 可见，可操作改为通过/删除 |

### 页面交互细节

**学生"我的作品"页**：
- 每个文件卡片右上角显示状态标签：
  - 待审核：黄色标签「待审核」
  - 已通过：绿色标签「已通过」
  - 已拒绝：红色标签「作品被打回」
- 上传成功后提示文案由"上传成功！"改为"上传成功！作品已提交，请等待教师审核通过后展示给同学"

**教师"文件管理"页**：
- 表格新增"审核状态"列
- 待审核文件行显示操作按钮：「通过」「拒绝」
- 已拒绝文件行显示操作按钮：「改为通过」（允许教师改变决定）
- 可按审核状态筛选文件

### 删除逻辑说明

无论文件处于何种审核状态（待审核/已通过/已拒绝），学生均可在"我的作品"页面删除自己上传的作品。删除后文件从服务器物理删除、数据库记录清除。

## 变更二：学生删除自己作品（已实现，确认保留）

### 当前实现

学生"我的作品"页面，鼠标悬停在文件卡片上时，右上角出现 🗑 删除按钮。点击后弹出确认框，确认后通过 `DELETE /file/<id>` 接口删除。

### 权限校验

```python
# 学生只能删自己的文件
if session.get('role') != 'admin' and record.uploader_id != session['user_id']:
    return jsonify({'error': '无权删除此文件'}), 403
```

### 保持不变

此功能无需改动，与新增的上传审核机制兼容。学生可删除自己任何状态（待审核/已通过/已拒绝）的作品。
